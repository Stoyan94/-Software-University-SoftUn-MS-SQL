агрегиращите функции винаги връщат един единствен резултат за целия result set, но result set може да не е един, ние може да гропираме и да направим X групи които са отделни result sets, съответно ще получим X резултата, това не нарушава правилото, че агрегиращата функия връщат един единствен резултат и трябва да запомни, че когато искаме да напишем нещо като 'WHERE CLAUSE' в заявка която ползваме 'GROUP BY' и агрегиращи функции и целта ни е да получив повече от една група трябва да изпозваме HAVING

aggregate functions always return a single result for the entire result set, but the result set may not be one, we can group and make X groups which are separate result sets, accordingly we will get the X result, this does not break the rule , that the aggregate function returns a single result and you have to remember that when you want to write something like 'WHERE CLAUSE' in the query you use 'GROUP BY' and aggregate functions and the goal is not to get more than one group you have to use HAVING
